
# 抖音猜词直播小游戏 -- 技术方案

## 一、整体架构概览

```
观众手机(抖音小程序) --WebSocket--> 后端服务(云服务器) --Redis--> 排行榜
                                    |
                                    +--> 语义匹配引擎(Embedding + 向量检索)
                                    |
                                    +--> 词库管理 + 游戏轮次管理
主播端(OBS/推流) <--WebSocket-- 排行榜展示页面(H5大屏)
```

核心流程：
1. 主播开启一轮游戏，系统展示提示词/线索
2. 观众在抖音小程序中输入猜测的词
3. 后端对输入做语义相似度匹配（与标准答案对比）
4. 按相似度排名，实时刷新排行榜
5. 主播端的排行榜 H5 页面通过 WebSocket 实时更新

### 2.1 服务器资源

| 资源 | 推荐 | 月费(约) | 说明 |
|------|------|----------|------|

### 2.2 域名与SSL

| 资源 | 说明 | 费用 |
|------|------|------|
| 域名 | 在阿里云/腾讯云注册一个 .com 或 .cn 域名 | 约30-60元/年 |

**注意**：抖音小程序要求后端域名必须 HTTPS 且已备案，这是硬性前提。

### 2.3 抖音开放平台资质

| 资源 | 说明 |
|------|------|
| 抖音开放平台账号 | https://open.douyin.com 注册开发者账号 |

### 2.4 AI/语义匹配资源

| 方案 | 说明 | 费用 |
|------|------|------|
| 方案A: 硅基流动/智谱AI API | 调用云端 Embedding 模型（如 bge-large-zh） | 按量付费，小规模几乎免费 |
| 方案B: 本地部署 | 在ECS上跑轻量 Embedding 模型（如 text2vec-chinese） | 需GPU，成本高不推荐 |
| **推荐方案A** | 调用硅基流动 SiliconFlow API，bge-m3 模型，免费额度充足 | 免费/极低 |

### 2.5 总成本估算

**首月总计：约 100-200 元**（含服务器 + 域名 + 免费额度内的API调用）

## 三、技术栈选型

### 3.1 后端

| 组件 | 技术 | 说明 |
|------|------|------|
| 语言/框架 | **Node.js + Express/Fastify** 或 **Python + FastAPI** | 推荐 Node.js，前后端统一语言 |
| 实时通信 | **WebSocket (ws/Socket.IO)** | 排行榜实时推送 |
| 数据库 | **PostgreSQL** 或 **SQLite**（小规模） | 存储词库、用户、游戏记录 |
| 缓存/排行 | **Redis** (Sorted Set) | 实时排行榜天然适合 Redis ZSET |
| 语义匹配 | **Embedding API + 余弦相似度** | 对比观众输入与答案的语义距离 |

### 3.2 前端

| 组件 | 技术 | 说明 |
|------|------|------|
| 观众端 | **抖音小程序**（TikTok Mini App） | 用抖音的 DSL 开发，类似微信小程序 |
| 主播大屏 | **React + Vite** H5 页面 | OBS 浏览器源捕获，展示排行榜 |
| 后台管理 | **React + Ant Design** | 词库管理、游戏配置（可后期加） |

### 3.3 语义匹配（RAG简化版）

本项目不需要完整的 RAG 管线，核心是 **Embedding + 相似度计算**：

```
观众输入 "苹果手机"
         |
    Embedding API (bge-m3)
         |
    得到向量 [0.12, -0.34, ...]
         |
    与答案 "iPhone" 的 Embedding 向量
    计算余弦相似度 -> 0.87
         |
    相似度 > 阈值(如0.75) -> 判定为猜对
```

不需要向量数据库（小规模），直接在内存中计算即可。如果词库超过1000条，可以用简单的 numpy 向量检索。

## 四、核心模块设计

### 4.1 游戏轮次管理

```
每轮游戏状态:
- word_id: 当前目标词
- hint: 提示线索
- answer_embedding: 答案的向量（预计算）
- start_time / end_time
- status: waiting | active | finished

流程:
1. 主播点击"开始新一轮" -> 从词库随机/指定选词
2. 预计算答案 Embedding
3. 广播给所有观众端"新一轮开始 + 提示词"
4. 接收观众猜测，计算相似度，写入 Redis ZSET
5. 倒计时结束 -> 公布答案，展示最终排行
```

### 4.2 排行榜（Redis ZSET）

```
Key: game:{round_id}:leaderboard
Score: 相似度 * 1000 (保留3位精度)
Member: user_id + guess_text

Redis 命令:
ZADD game:1:leaderboard 872.5 "user123:苹果手机"
ZREVRANGE game:1:leaderboard 0 49  -- 取Top50
```

### 4.3 WebSocket 通信协议

```json
// 服务端 -> 客户端: 新一轮开始
{ "type": "new_round", "data": { "roundId": 1, "hint": "一种水果", "duration": 60 }}

// 客户端 -> 服务端: 提交猜测
{ "type": "guess", "data": { "roundId": 1, "text": "苹果" }}

// 服务端 -> 客户端: 排行榜更新 (每秒/每5秒推送)
{ "type": "leaderboard", "data": { "roundId": 1, "entries": [{"rank":1,"user":"xxx","score":0.95,"text":"苹果"}] }}

// 服务端 -> 客户端: 本轮结束
{ "type": "round_end", "data": { "answer": "苹果", "winner": "user123" }}
```

### 4.4 抖音弹幕获取（备选方案）

如果小玩法审核未通过，备选方案是接入抖音弹幕：
- 通过抖音开放平台的"直播互动"能力，订阅 `live_comment` 事件
- 观众发评论"我猜XX"，后端解析评论提取猜测词
- 这种方式用户体验不如小程序，但技术门槛更低

## 六、词库设计

初始词库建议包含以下分类，每类 50-100 个词：

| 分类 | 示例 | 难度 |
|------|------|------|
| 水果 | 苹果、香蕉、草莓 | 简单 |
| 动物 | 老虎、企鹅、蝴蝶 | 简单 |
| 电影 | 肖申克的救赎、泰坦尼克号 | 中等 |
| 成语 | 画蛇添足、对牛弹琴 | 中等 |
| 科技 | 人工智能、区块链 | 较难 |

提示词示例：
- 目标词"苹果" -> 提示："一种常见的红色水果，也是一家科技公司的名字"
- 目标词"画蛇添足" -> 提示："比喻多此一举的成语，和动物有关"

## 七、部署与上线流程

### Task 1: 环境准备（第1天）
1. 购买云服务器（推荐腾讯云轻量 2核4G）
2. 注册域名，完成 ICP 备案（1-2周，先启动）
3. 安装 Docker、Node.js、Redis、Nginx
4. 配置 SSL 证书（Let's Encrypt）

### Task 2: 后端开发（第2-5天）
1. 初始化 Node.js + TypeScript 项目
2. 实现游戏轮次管理（创建/开始/结束）
3. 实现 WebSocket 服务（连接管理、消息广播）
4. 实现排行榜逻辑（Redis ZSET）
5. 接入 Embedding API（硅基流动 SiliconFlow）
6. 实现语义匹配 + 相似度排名
7. 词库管理 CRUD + 初始数据导入

### Task 3: 主播端 H5 开发（第6-7天）
1. React + Vite 初始化
2. WebSocket 连接 + 实时排行榜渲染
3. 游戏状态展示（倒计时、当前提示、轮次结果）
4. 动画效果（排名变动动效、猜对庆祝）

### Task 4: 抖音小程序开发（第8-12天）
1. 注册抖音开放平台，创建小程序应用
2. 开发猜词页面（输入框 + 提交 + 排行榜）
3. 接入抖音登录（获取用户头像昵称）
4. WebSocket 连接后端
5. 提交审核

### Task 5: 部署上线（第13-14天）
1. docker-compose 打包部署
2. Nginx 反向代理配置
3. 域名解析 + HTTPS
4. 测试全流程

### Task 6: 直播测试（第15天）
1. OBS 添加浏览器源（排行榜H5页面）
2. 小范围直播测试
3. 根据反馈调整

## 八、关键注意事项

1. **ICP 备案先行**：备案需要1-2周，建议第一天就提交，否则后续全部卡住
2. **抖音小玩法审核**：小程序上线需要审核，建议提前了解审核规范，预留3-5天
3. **Embedding 缓存**：词库中所有答案的 Embedding 应预计算并缓存，避免每次实时调用API
4. **防刷机制**：限制每个用户每轮猜测次数（如最多3次），防止刷屏
5. **弹幕备选**：如果小程序审核不通过，可以先用弹幕方案快速验证
