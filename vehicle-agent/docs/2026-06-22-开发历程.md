## 一、高德地图集成

### 1. 背景

之前使用的是开源地图（Leaflet/OpenStreetMap），功能和体验有限。用户要求替换为**高德地图**，支持真实道路渲染和 GPS 导航。

### 2. 实现内容

- 安装 `@amap/amap-jsapi-loader` 官方 SDK
- 后端新增 `/api/vehicle/agent-info` 接口，返回高德地图 JS API Key 和 SecurityCode
- `MapPanel.tsx` 全面重写，使用 `AMap.Map` + `AMap.Driving` 插件

### 3. 踩坑：坐标格式不匹配

**现象**：地图只显示蓝色定位点，没有道路和建筑物，路线也不显示。

**根因**：高德地图 API 使用 `[lng, lat]`（经度在前），而项目内部坐标统一使用 `[lat, lng]`（纬度在前）。所有传给高德 API 的坐标都需要转换。

**修复**：添加系统性坐标转换函数：

```tsx
// [lat, lng] → 高德 AMap.LngLat 对象（内部是 lng,lat 顺序）
function toLngLat(coord: [number, number], AMap: any): any {
  return new AMap.LngLat(coord[1], coord[0]);
}

// [lat, lng] → 高德数组格式 [lng, lat]
function toAmapArray(coord: [number, number]): [number, number] {
  return [coord[1], coord[0]];
}
```

所有传给高德 API 的坐标（地图中心、Marker 位置、Polyline 路径、Driving 起终点）都经过转换。

### 4. 踩坑：AMap 全局对象重复初始化

**现象**：地图偶尔白屏，控制台报 "AMap is not defined"。

**根因**：`(window as any).AMap` 在某些生命周期阶段不可靠，且同一 Key 重复调用 `AMapLoader.load()` 会出错。

**修复**：用 `useRef` 缓存 AMap 全局对象 + `initKeyRef` 防止重复初始化：

```tsx
const AMapRef = useRef<any>(null);
const initKeyRef = useRef("");

if (initKeyRef.current === amapKey && mapRef.current) return; // 跳过重复初始化
AMapLoader.load({ key: amapKey, ... }).then((AMap) => {
  AMapRef.current = AMap;
  initKeyRef.current = amapKey;
});
```

### 5. 路线渲染策略

```
有起点+终点 → AMap.Driving 插件规划真实路线（显示道路、转弯）
仅有 routeCoords → 手动 Polyline 折线（fallback）
POI 标记 → 橙色圆点 + 名称标签
```

## 二、起点选择交互

### 1. 需求

当用户说"导航去公司"但没指定起点时，Agent 需要调用 `select_origin` 工具，提供选项卡片（家、公司、火车站、机场），让用户点选出发地点。

### 2. 起点选择 UI 方案演进

| 版本 | 方案 | 问题 |
|------|------|------|
| v1 | `useFrontendTool` + OriginSelector 弹框 | Agent 调了 `select_origin` 但弹框不弹出 |
| v2 | OriginSelector 用 React Portal 渲染到 body | 弹框能弹出，但按钮点击无效 |
| v3 | `useHumanInTheLoop` 在聊天内联渲染按钮 | 按钮显示为纯文字，无法点击 |
| v4 | `useRenderTool` + 外部 Portal 弹框 | 类型不兼容，`useRenderTool` 不支持 handler 回传结果 |
| **v5（最终）** | 自定义可折叠聊天面板 + `useHumanInTheLoop` | ✅ 选择卡片正常渲染和交互 |

## 三、前端工具注册机制深度攻坚

### 1. 核心问题

`update_map` 和 `select_origin` 是前端工具，但在 Supervisor 多 Agent 架构下，子 Agent 看不到这些工具：

```
前端 useFrontendTool("update_map")
    ↓ AG-UI 协议发送工具定义到后端
    ↓ state["tools"] 里有 update_map
    ↓
❌ navigation_agent 创建时只绑定 MCP 工具，不看 state["tools"]
    ↓ LLM 看不到 update_map，不会调用
```

### 2. 错误尝试

- **后端 StructuredTool stub**：在后端添加 `update_map`/`select_origin` 的占位工具 → stub 直接执行返回占位值，前端 handler 从未被调用
- **`useRenderTool` 监听**：只监听工具执行事件做副作用，不能让 LLM 主动调用前端工具

### 3. 关键发现：CopilotKitMiddleware

调研 CopilotKit 官方文档，发现 **`CopilotKitMiddleware`** 是官方推荐的前端工具注入方案：

> "CopilotKitMiddleware is what forwards it — drop it into your create_agent call"

**工作原理**：
```
前端 useFrontendTool / useHumanInTheLoop
    ↓ AG-UI 协议发送工具定义
    ↓
CopilotKitMiddleware 自动拦截
    ↓ 将前端工具注入到 Agent 的工具列表中
    ↓ LLM 能看到并调用前端工具
    ↓ 调用时 AG-UI 路由到前端 handler 执行
```

**关键限制**：
- `create_agent`（langchain.agents）支持 `middleware` 参数 ✅
- `create_react_agent`（langgraph.prebuilt）**不支持** middleware 参数 ❌

### 4. 最终实现

**后端 `navigation_agent.py`**：
```python
from copilotkit import CopilotKitMiddleware

return create_agent(
    model=create_llm(),
    tools=navigation_tools,
    name="navigation_agent",
    system_prompt=NAVIGATION_PROMPT,
    middleware=[CopilotKitMiddleware()],  # 自动注入前端工具
)
```

**后端 `supervisor.py`**：移除 `frontend_bridge_tools` 的 import 和手动注入逻辑。

**前端 `App.tsx`**：
- `select_origin` → `useHumanInTheLoop`（Agent 暂停等待用户选择）
- `update_map` → `useFrontendTool`（前端 handler 执行并返回结果）

### 5. 为什么 `useHumanInTheLoop` 在 CopilotSidebar 里按钮不可点击

**现象**：`useHumanInTheLoop` 的 `render` 函数返回的选择卡片在 CopilotSidebar 聊天中显示为纯文字，按钮无法点击。

**原因分析**：CopilotSidebar 的聊天气泡 DOM 结构和 CSS 样式限制了内部元素的交互能力（可能是 `pointer-events` 层叠或事件拦截）。

**最终解决**：弃用 CopilotSidebar，改用 `CopilotChat` 组件嵌入自定义的可折叠右侧面板中。`CopilotChat` 是干净的聊天区域，没有侧边栏包裹层的 CSS 限制，`useHumanInTheLoop` 的选择卡片能正常渲染和交互。

## 六、关键知识点总结

### 1. CopilotKit 前端工具注入的正确方式

| 场景 | 方案 | 是否需要后端 stub |
|------|------|-----------------|
| 单 Agent + 前端工具 | `useFrontendTool` 直接用 | ❌ 不需要 |
| 多 Agent + 前端工具 | `CopilotKitMiddleware` + `create_agent` | ❌ 不需要 |
| 需要用户交互 | `useHumanInTheLoop` + `CopilotKitMiddleware` | ❌ 不需要 |
| 只监听后端工具执行 | `useRenderTool` | ✅ 需要后端工具 |

### 2. 高德地图集成要点

- 高德 API 坐标格式：`[lng, lat]`（经度在前）
- 安全密钥：`window._AMapSecurityConfig = { securityJsCode: ... }`（必须在 `AMapLoader.load` 之前设置）
- Driving 插件：`new AMap.Driving({ map, policy: AMap.DrivingPolicy.LEAST_TIME })`
- 不要重复 `AMapLoader.load()`，用 ref 缓存

### 3. CopilotChat vs CopilotSidebar

| 特性 | CopilotSidebar | CopilotChat |
|------|---------------|-------------|
| 渲染方式 | 固定侧边栏，包裹页面 | 内嵌组件，可放在任意容器 |
| CSS 隔离 | 有（可能限制内部交互） | 较干净 |
| useHumanInTheLoop 交互 | ⚠️ 按钮可能不可点击 | ✅ 正常 |
| 自定义折叠控制 | 不支持 | ✅ 支持 |

### 4. `create_agent` vs `create_react_agent`

| 特性 | `create_agent` | `create_react_agent` |
|------|---------------|---------------------|
| 来源 | `langchain.agents` | `langgraph.prebuilt` |
| middleware 支持 | ✅ | ❌ |
| CopilotKitMiddleware | ✅ 可用 | ❌ 不可用 |
| 适用场景 | 需要前端工具注入 | 纯后端工具 |
