# A2 迁移 + 前端交互优化 实施计划

## 目标（用户 5 项需求）
1. 后端架构改为 **方式 A2**：放弃 `langgraph-supervisor`，改用官方 Sub-Agents 模式（supervisor = `create_agent` + `CopilotKitMiddleware`，每个子 Agent 包成 `@tool` 同步 invoke）。
2. 前端起点选择**不再弹模态框**，改为聊天对话流中的**小卡片**。
3. 导航的超长文字结果（逐条路线步骤）**不显示在聊天里**。
4. 历史调用的各 Agent 结果**保留不清除**，可查看历史调用记录。
5. 修复"刷新导航路线很慢"的问题。

---

## 一、后端 A2 改造

### 1. `app/agents/*.py` — 子 Agent 保持 `create_agent`，去掉 CopilotKitMiddleware
- `navigation_agent.py`：删除 `middleware=[CopilotKitMiddleware()]`，只保留 MCP 工具（plan_route/search_poi/traffic）。其余子 Agent 不变。
- 每个 `create_*_agent()` 已返回可 `ainvoke` 的图，无需改动签名。

### 2. 新增 `app/graph/subagent_tools.py` — 把子 Agent 包成 @tool
- 对 navigation/media/vehicle/weather/reminder 各生成一个 `@tool`。
- 工具内部：构造 `{"messages":[HumanMessage(task)]}` → `await sub_agent.ainvoke(...)` → 取最后一条 AIMessage 文本作为 ToolMessage 返回。
- 子 Agent 内部消息**不冒泡**到 supervisor，杜绝孤儿 tool_call。

### 3. 重写 `app/graph/supervisor.py`
- supervisor = `create_agent(model, tools=[...子Agent工具], system_prompt=_build_prompt, middleware=[CopilotKitMiddleware()], state_schema=AutoMindState)`。
- `select_origin` / `update_map` 前端工具由 CopilotKitMiddleware 在 supervisor 层自动注入。
- HITL 暂停发生在 supervisor 单一作用域 → `_validate_chat_history` 不再报错。
- 编排流程（写进 supervisor prompt）：
  1. 用户未给起点 → supervisor 先调 `select_origin`（HITL，在顶层暂停/恢复）。
  2. 调 `navigation_agent` 工具（传 from/to）拿到路线摘要 + 结构化数据。
  3. 调 `update_map` 把路线画到地图。
  4. 用一句话简短播报（不复述逐条步骤）。
- 删除 `langgraph_supervisor` 依赖；`routing.py` 的 `route_intent` 不再需要（保留描述文本注入 prompt）。

### 4. 性能：精简 update_map 负载
- `navigation_tools.plan_route`：对 `route_coords` **降采样到 ≤40 个点**（保留首尾），大幅减少 LLM 流式输出体量。
- supervisor prompt 约定 `update_map` 只传 origin/destination/坐标/distance/duration，**不再把长 steps 塞进聊天**。

## 二、前端改造

### 5. `select_origin` 改内联小卡片（去模态）
- `App.tsx` 的 `useHumanInTheLoop({name:"select_origin", render})`：把 `HumanChoiceSelector`（`createPortal` 全屏模态）替换为**内联卡片** `OriginChoiceCard`，直接渲染在聊天气泡流中（不 portal、不全屏遮罩）。
- 卡片：紧凑的横向选项 chips + 可选自定义输入，点击后 `respond(value)`。
- complete 状态显示"✓ 已选择 家"小标签，**保留在历史里**。

### 6. 隐藏导航长文本 + 保留调用历史卡片
- `update_map`（`useFrontendTool`）增加 `render`：渲染**紧凑导航卡片**（🧭 起点→终点 · X km · Y 分钟），不显示逐条 steps；inProgress/complete 状态都给小卡片，保留在消息流。
- 用 `useDefaultRenderTool` 给 navigation_agent 等后端工具调用一个**通用紧凑卡片**（"🤖 导航助手 已完成"），避免大段原始文本刷屏；历史调用自然保留（v2 工具调用默认留在 thread 中）。

### 7. 修复刷新慢（MapPanel）
- 删除 [MapPanel.tsx](../frontend/src/components/MapPanel.tsx) 中 `drivingRef.current.search(...)` 的二次网络规划，**直接用后端返回的 `routeCoords` 画 Polyline**（后端已是高德真实路径）。
- 去掉多余的 `setCenter`+`setFitView` 竞争，统一一次 `setFitView`。
- 结果：地图刷新 = 本地直接画线，无第二次高德网络请求。

- 前端：`npm run build`（tsc + vite）通过。
- 手测：起点为内联卡片、聊天无长文本、历史卡片保留、地图秒刷。
