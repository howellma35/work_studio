# 2026-07-04 HTTPS 部署全链路调试日志

## 背景

项目采用内网穿透架构部署：家里 Docker 服务 → frp 隧道 → 阿里云 ECS → 公网访问。之前 HTTP 开发端口(6000/6001/6002)已通过 frp 穿透正常工作，但生产域名 `mahongwei.com.cn` 的 HTTPS (443) 访问一直不通。

## 今天解决的问题

### 问题 1：frpc token 与 frps 不一致

**症状**：
- `curl http://mahongwei.com.cn` 返回 Empty reply from server
- `curl https://mahongwei.com.cn` 返回 SSL connect error
- ECS 上 `sudo ss -tlnp | grep frps` 只看到 6000/6001/6002/7000/7500，没有 80/443

**根因**：frpc 日志明确报 `login to the server failed: token in login doesn't match token from configuration`，frpc 从未成功连接 frps，穿透规则从未注册。

**解决**：对比 `/etc/frp/frps.toml` 和 `/etc/frp/frpc.toml` 的 `auth.token`，改为一致后重启 frpc。

**经验**：修改 frpc 配置后只需重启 frpc，frps 无需重启（frps 自动处理新连接注册）。

---

### 问题 2：SSL 证书不存在导致 Docker nginx 反复重启

**症状**：
- `docker ps | grep nginx` 显示 `Restarting (1) 11 seconds ago`
- `docker logs nginx` 报 `cannot load certificate "/etc/letsencrypt/live/mahongwei.com.cn/fullchain.pem": No such file or directory`

**根因**：家里机器从未申请过 SSL 证书，但 nginx.conf 同时配了 HTTP 和 HTTPS（引用证书文件），找不到文件导致 nginx 启动失败。

**决策**：将 SSL 终结从家里 nginx 移到 ECS nginx，这样：
- 证书在 ECS 上申请，certbot standalone 直接验证，不走 frp 隧道
- 家里 nginx 只做 HTTP，不需要证书
- 架构更清晰

**新架构**：
```
用户 → ECS nginx (443 SSL终结) → 127.0.0.1:6003 → frps → frp隧道 → 家里 nginx:80
```

---

### 问题 3：nginx.conf 文件内容重复

**症状**：写入新配置时旧内容没有被覆盖，导致文件中出现两套 server block（旧的 HTTPS + 新的 HTTP-only）。

**解决**：用完整正确的 HTTP-only 配置重写整个文件。

---

### 问题 4：ECS nginx 与 frps 的 80 端口冲突

**症状**：
- ECS nginx 需要监听 80（HTTP→HTTPS 重定向）
- frps 的 nginx-http 穿透规则 `remotePort=80` 也在 ECS 上监听 80
- 两者冲突，nginx 无法启动

**解决**：
- frpc 穿透规则的 `remotePort` 从 80 改为 6003
- ECS nginx 的 `proxy_pass` 改为 `http://127.0.0.1:6003`
- 删除 frpc 中的 `nginx-https`（443）穿透规则（不再需要）

**额外发现**：ECS nginx proxy_pass 到 127.0.0.1:6003 是本机内部通信，UFW 和阿里云安全组**不需要放行 6003 端口**（之前文档里写了放行 6003 是多余的，已删除）。

---

### 问题 5：docker-compose.yml 端口映射未更新

**症状**：`docker compose up -d nginx` 后容器仍映射 443 端口。

**分析**：
- docker-compose.yml 已改为只映射 80，但家里的文件可能没同步
- 即使文件同步了，`docker compose up -d` 默认不重建已有容器（端口映射变更需要 `--force-recreate`）
- 实际上 443 映射不影响功能（nginx 内部不监听 443 就不会有流量进来）

**解决**：同步最新文件后 `docker compose up -d --force-recreate nginx`。

---

### 问题 6：ECS nginx 443 端口 Connection refused

**症状**：证书存在、配置语法通过（`nginx -t` OK），但 `sudo ss -tlnp | grep nginx` 只有 80，没有 443。

**根因**：nginx 启动时 frps 还占着 80 端口（旧穿透规则 remotePort=80），nginx 启动过程中 80 端口绑定失败，导致整个进程异常，HTTPS server block 没有生效。

**解决**：
1. 先修改 frpc 的 remotePort=6003，重启 frpc 释放 80 端口
2. 再 `sudo systemctl restart nginx`，443 端口正常监听

---

### 问题 7：UFW/安全组配置清理

发现文档中有多处多余的安全组规则：
- 6003 端口不需要放行（本机内部通信）
- 443 穿透规则已删除，不需要放行 443 给 frps

已清理文档，保留正确规则：80/443（ECS nginx）、7000（frps）、6000/6001/6002（开发调试）、7500（Dashboard）。
