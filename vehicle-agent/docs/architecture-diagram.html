Supervisor 工作流程

🔄 动态提示词构建
_build_prompt(state) 每次调用时动态注入：
① 路由描述（5个子Agent能力说明）
② 用户档案（SQLite 结构化数据）
③ 召回偏好（ChromaDB 语义检索 top-3）
④ 待处理提醒（SQLite 查询）
⑤ 完整对话历史（messages 列表）
🎯 路由决策机制
create_supervisor() 内部使用 LLM 决策：
① Supervisor LLM 分析用户意图
② 输出 transfer_to_xxx 工具调用
③ 子Agent 通过 create_react_agent 执行
④ 子Agent 完成后自动返回 Supervisor
⑤ Supervisor 决定继续委派或结束
⑥ MemorySaver 保存每步 checkpoint

MCP (Model Context Protocol) 工具体系

基于 FastMCP 构建的统一工具服务器，通过 langchain-mcp-adapters 桥接到 LangChain

🔌 MCP 连接架构

连接模式
stdio 模式（开发推荐）
Agent 启动 → MultiServerMCPClient → 创建子进程 python -m app.mcp.server

SSE 模式（生产部署）
Agent 启动 → MultiServerMCPClient → HTTP 连接 http://localhost:8765
工具加载流程
1. load_mcp_tools() 创建 MCP Client
2. 根据 MCP_TRANSPORT 选择 stdio/SSE
3. 调用 client.get_tools() 获取所有工具
4. 返回 list[BaseTool] LangChain 标准工具
5. 各子Agent 按关键词过滤分配工具

📦 MCP Server 聚合架构

server.py 主服务器
FastMCP("AutoMindVehicleServer")

聚合 4 个子工具集：
├── navigation_tools → FastMCP("NavigationTools")
│   ├── plan_route(origin, destination, avoid_traffic)
│   ├── search_poi(keyword, location)
│   └── get_traffic_info(route_id)
├── media_tools → FastMCP("MediaTools")
│   ├── play_music(song_name, artist)
│   ├── pause_music()
│   ├── next_song()
│   ├── set_volume(level)
│   └── get_playlist()
├── vehicle_tools → FastMCP("VehicleTools")
│   ├── control_window(position, action)
│   ├── set_climate(temp, mode, fan_speed)
│   ├── lock_doors(action)
│   ├── set_seat(position, heated)
│   └── get_vehicle_status()
└── weather_tools → FastMCP("WeatherTools")
    ├── get_weather(city)
    └── get_forecast(city, days)

工具注册机制

每个工具模块创建独立的 FastMCP 实例，使用 @mcp.tool() 装饰器注册工具。 server.py 在启动时 import 所有工具模块，通过遍历 _tool_manager._tools 将子模块工具合并到主服务器。 这种设计实现了工具的模块化开发与统一暴露。

reminder_agent 的特殊之处：不使用 MCP 工具，而是通过 @tool 装饰器定义内置 LangChain 工具， 直接操作 memory_manager 实现提醒的 CRUD 操作。

记忆系统架构

双层记忆：短期对话 + 长期偏好，通过 MemoryManager 统一对外暴露

⏱️ 短期记忆 (Short-Term)

💬
MemorySaver
实现: langgraph.checkpoint.memory.MemorySaver
存储: 内存（进程内）
机制: LangGraph checkpointer
作用: 多轮对话状态持久化
会话隔离: 通过 thread_id 区分
特性: 支持状态时间旅行调试

生产替换:
→ PostgresSaver (PostgreSQL)
→ RedisSaver (Redis)

🧬 长期记忆 (Long-Term)

🔍
向量记忆 · ChromaDB
用途: 用户偏好语义存储与召回
嵌入模型: text-embedding-v3 (百炼平台)
持久化: ./data/chroma
集合: user_preferences

写入: save_preference() → 生成 embedding → ChromaDB add
召回: recall_preferences() → query embedding → top-3 相似
元数据: user_id + category 过滤
📋
结构化记忆 · SQLite
数据库: ./data/memory.db

user_profiles 表:
user_id (PK) | profile (JSON) | updated_at
→ get_profile() / update_profile()

reminders 表:
id (PK) | user_id | content | remind_at | is_done | created_at
→ add_reminder() / get_reminders()

MemoryManager 统一接口

get_context(user_id, query)
① recall_preferences(user_id, query) → 语义召回
② get_profile(user_id) → 结构化档案
③ get_reminders(user_id) → 待处理提醒
→ 合并为上下文 dict 注入 Supervisor prompt
update_profile(user_id, updates)
① SQLite: INSERT OR UPDATE user_profiles
② ChromaDB: 每条更新 → save_preference()
→ 双写保证一致性
add_reminder(user_id, content, remind_at)
① 生成 UUID 短 ID
② SQLite INSERT INTO reminders
→ 返回 reminder_id
