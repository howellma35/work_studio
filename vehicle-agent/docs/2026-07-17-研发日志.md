## 二、审计结论（详见 `docs/2026-07-17-官方文档合规优化方案.md`）

### 已合规（保持不动）

- **多 Agent 编排**：supervisor = `create_agent(middleware=[CopilotKitMiddleware()])`，5 个子 Agent 包成 `@tool` 经 `subagent_tools.py` 同步 `ainvoke`，子 Agent 内部消息不冒泡 → 无孤儿 tool_call / INVALID_CHAT_HISTORY。**这正是 CopilotKit 官方 Sub-Agents canonical 写法。**
- **子 Agent 不挂 CopilotKitMiddleware**（官方："intentionally do NOT attach on sub-agents"）。
- 前端 `useFrontendTool` / `useHumanInTheLoop` 按工具名匹配、未用 `agentId`。
- `AutoMindState(CopilotKitState)`、`AsyncSqliteSaver` 持久 checkpointer、`@lru_cache(maxsize=8)`、`pydantic-settings` 配置、`create_agent`+`system_prompt`、`add_langgraph_fastapi_endpoint`+`LangGraphAGUIAgent`。

### 关于"HITL prompt 在后端是否正确"的关键澄清

要区分两层：

| 层 | 该在哪 | 当前状态 |
|----|--------|----------|
| 工具定义（name/description/parameters/render） | **前端** `useHumanInTheLoop({...})`，经 AG-UI `copilotkit.actions` 传输，`CopilotKitMiddleware` 注入 | ✅ 已对（后端不定义工具） |
| supervisor 系统提示词（何时用） | **后端**（Agent 的大脑，官方允许） | ✅ 正确，但**夹带了前端工具的参数硬编码**（options 默认值、update_map 字段逐字罗列）→ 违背"工具参数 schema 归前端单源" |

结论：后端有系统提示词本身正确；问题是它把本该属于前端工具 schema 的默认值/字段列表也抄了一份，造成三处漂移。本轮做了单源化修正。

## 四、第二轮改动（本轮，官方最佳实践对齐）

### D1. HITL 工具单源化（`frontend/src/App.tsx`）
- `select_origin` 的 `options`/`message` 加 zod `.default([...])`，**工具 schema 自带默认值**。
- 后端 prompt 不再硬编码 `options=["家","公司","火车站","机场"]`。
- `CopilotChat.instructions` 精简：删除与后端 prompt 重复的工具调用规则，并删除**过时的** `plan_route`/`search_poi` 直调引用（A2 下 supervisor 调 `navigation_agent`，并不直接调子 Agent 内部 MCP 工具），仅保留"只支持导航功能"产品声明。

### D2. 后端 supervisor prompt 精简（`graph/supervisor.py`）
- 去掉硬编码 `options`/`message` 值（已移入前端 zod）。
- `update_map` 不再逐字罗列全部字段，改为行为性引导"把起点/终点/坐标/距离/时长传给 update_map"。
- 保留 `ROUTE_DATA`/`POI_DATA` 子 Agent 数据契约（这是 supervisor 与 navigation_agent 之间的后端内部契约，非前端工具 schema 重复，必须保留）。

### D3. MCP 改为单一 FastMCP 实例（官方写法）
- `mcp/tools/__init__.py`：定义共享 `mcp = FastMCP("AutoMindVehicleServer")`。
- 4 个工具模块（navigation/media/vehicle/weather）改为 `from app.mcp.tools import mcp`，不再各自 `FastMCP(...)`。
- `mcp/server.py`：删除私有属性 `_tool_manager._tools` 合并循环，改为导入子模块触发注册后 `mcp.run()`。
- 静态分析确认无循环导入；`py_compile` 通过。

### D4. 补缺失依赖（`requirements.txt`）
- 新增 `ag-ui-langgraph>=0.1.0`——`main.py` 已 `from ag_ui_langgraph import ...` 但 requirements 未声明，新装环境会 ImportError。真实打包 bug。
