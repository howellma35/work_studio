# 2026-07-04 服务地址映射手册

> 本文档梳理了所有服务的访问地址，分三种场景：家里内网直连、ECS 公网 IP、域名访问。

## 网络架构概览

```
公网用户 → ECS (106.14.24.38) → frp 隧道 → 家里服务器 (192.168.31.101)

ECS 上有：
  - nginx: SSL 终结 (80/443) + HTTP→HTTPS 重定向
  - frps: 内网穿透服务端，监听各穿透端口

家里有：
  - Docker 全部服务
  - frpc: 内网穿透客户端
  - nginx: HTTP 反向代理 (80端口)
```

## 四、Langfuse 可观测性平台

> 家里服务器 32GB 内存，可以正常启用 Langfuse。Langfuse 随默认启动（无需 `--profile langfuse`）。

### 启动 Langfuse

```bash
cd /data/caragent/work_studio/deploy
docker compose up -d --build   # Langfuse 默认启动
```

### Langfuse 访问地址

| 场景 | 地址 | 推荐 |
|------|------|------|
| 家里内网直连 | http://192.168.31.101:3000 | ✅ 推荐 |
| SSH 端口转发 | `ssh -L 3000:localhost:3000 devon@192.168.31.101` 然后浏览器打开 http://localhost:3000 | ✅ 推荐（从外网访问时） |
| 域名访问 | https://mahongwei.com.cn/langfuse/ | ⚠️ 不推荐（见下方说明） |
| frp 穿透端口 | http://106.14.24.38:6003 | ⚠️ 不推荐（同上） |

### ⚠️ 为什么不建议将 Langfuse 暴露到公网域名

**安全风险**：
1. Langfuse 默认开启注册（`LANGFUSE_ENABLE_SIGNUP: "true"`），任何人访问域名都能注册账号
2. 注册后能看到所有 LLM 调用日志，包括 prompt 内容、API Key、用户对话等敏感数据
3. 即使关闭注册，Langfuse 面板也不应对外暴露（管理工具应限制在内网）

**推荐做法**：
- 在家里或公司，直接通过内网 IP 访问：`http://192.168.31.101:3000`
- 从外网访问时，用 SSH 端口转发：`ssh -L 3000:localhost:3000 devon@192.168.31.101`，然后浏览器打开 `http://localhost:3000`

### 如果一定要通过域名访问（不推荐）

需要配置三层穿透 + 认证保护：

**第 1 步：添加 frp 穿透规则**（在家里 frpc.toml）

```toml
# 穿透 Langfuse（开发调试用，生产环境建议关闭）
[[proxies]]
name = "langfuse"
type = "tcp"
localIP = "127.0.0.1"
localPort = 3000
remotePort = 6004
```

**第 2 步：ECS 安全组放行**

阿里云安全组 + UFW 放行 6004 端口（建议限制为你的 IP）：
```bash
sudo ufw allow from 你的IP to any port 6004 proto tcp
```

**第 3 步：通过域名访问**

```bash
# 方式 1：ECS IP 直连
http://106.14.24.38:6004

# 方式 2：加到 ECS nginx（需额外配置 HTTP Basic Auth 保护）
# 不推荐，因为配置复杂且仍不安全
```

**第 4 步（必须）：关闭公开注册**

在 `deploy/.env` 中设置：
```ini
LANGFUSE_ENABLE_SIGNUP=false
```

然后重启：
```bash
docker compose up -d langfuse
```

> 关闭注册后，只有已注册的账号能登录，新访客无法使用。

### Langfuse 配置流程

**第 1 步：启动 Langfuse 后注册账号**

浏览器打开 `http://192.168.31.101:3000`，注册一个管理员账号。

**第 2 步：获取 API Key**

登录 Langfuse → Settings → API Keys → Create API Key，会得到：
- `LANGFUSE_PUBLIC_KEY`（pk-lf-xxx）
- `LANGFUSE_SECRET_KEY`（sk-lf-xxx）

**第 3 步：配置 .env 文件**

在 `deploy/.env` 中添加：

```ini
# Langfuse API keys（从 Langfuse 控制台获取）
LANGFUSE_PUBLIC_KEY=pk-lf-xxxxxxxx
LANGFUSE_SECRET_KEY=sk-lf-xxxxxxxx
LANGFUSE_HOST=http://192.168.31.101:3000
```

**第 4 步：在 AI 后端和车载助手后端集成 Langfuse**

在需要监控的后端代码中，添加 Langfuse SDK 调用：

```python
from langfuse import Langfuse

langfuse = Langfuse()
# 之后在 LLM 调用时自动追踪
```

**第 5 步：重启服务**

```bash
docker compose up -d --build ai-server vehicle-backend
```

## 六、请求流转路径

### HTTPS 请求（正常用户）

```
用户浏览器
  → https://mahongwei.com.cn/vehicle/
  → ECS nginx:443 (SSL 终结)
  → proxy_pass http://127.0.0.1:6003
  → frps:6003 → frp 隧道 → frpc
  → 家里 Docker nginx:80
  → location /vehicle/ → proxy_pass http://vehicle-frontend:80
  → 返回前端页面
```

### API 请求

```
用户浏览器
  → https://mahongwei.com.cn/api/copilotkit
  → ECS nginx:443 (SSL 终结)
  → proxy_pass http://127.0.0.1:6003
  → frps:6003 → frp 隧道 → frpc
  → 家里 Docker nginx:80
  → location /api/copilotkit → proxy_pass http://vehicle-runtime:4000
  → CopilotKit Runtime 处理
```

### 开发调试请求（ECS IP 直连）

```
开发者
  → http://106.14.24.38:6001/api/vehicle/health
  → ECS frps:6001 → frp 隧道 → frpc
  → 家里 192.168.31.101:8001 (vehicle-backend 直连)
  → 返回响应
```
