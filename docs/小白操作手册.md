## 第一部分：搞懂这些东西是什么

### 1.1 整个游戏是怎么工作的？

```
观众的手机                  你的电脑                   阿里云服务器(Ubuntu)
    │                        │                          │
    │  观众在抖音小程序       │  你打开排行榜网页        │
    │  里输入猜的词 ──────────────────────────────→ 后端程序接收
    │                        │                          │ 用阿里大模型算相似度
    │                        │  排行榜实时更新 ←────────── 返回排名
    │                        │                          │
    │  观众在自己手机上       │  OBS 把排行榜画面        │
    │  看到排行榜 ─────────── │  "拍"下来推给抖音        │
```

**用人话说就是（简化版）：**
- 阿里云服务器上跑着一个程序（后端），负责接收观众的猜测、用阿里大模型计算排名
- 你电脑上打开一个网页（排行榜），实时显示排名
- OBS 软件把这个排行榜网页的画面"拍"下来，放到你的直播画面里
- 观众在抖音直播间就能看到排行榜了

### 1.2 名词解释

| 名词 | 解释 |
|------|------|
| **服务器** | 一台 24 小时开着的远程电脑。你的程序放在上面一直运行，观众的手机才能随时访问。你自己电脑关了就没了。 |
| **Ubuntu** | 服务器的操作系统（就像你电脑装 Windows，服务器装 Ubuntu）。最常用的 Linux 发行版，社区大、教程多。 |
| **域名** | 网址，比如 `baidu.com`。服务器有一串数字 IP（比如 `47.100.1.234`），人记不住，所以用域名代替。 |
| **ICP 备案** | 国家规定：中国大陆的服务器必须登记你的身份信息。就像开店要办营业执照。**不备案 = 域名不能用**。免费，但要等 1-2 周。 |
| **SSL 证书** | 让网址从 `http://` 变成 `https://`（多了 s = 安全）。抖音要求必须 https。**免费的**。 |
| **OBS** | 免费直播软件，所有主播都在用。把你电脑上的画面（摄像头 + 排行榜网页）组合在一起，发送给抖音。 |
| **推流 / 推流地址** | "推流"就是把直播画面发送到抖音。"推流地址"是抖音给你的一个收货地址，OBS 把画面发到这个地址。 |
| **API Key** | 一把"钥匙"。程序要用阿里大模型算相似度，需要 API Key 证明你是合法用户。 |
| **SSH** | 远程操控服务器的方式。就像"远程桌面"，让你在本地电脑上输入命令控制服务器。 |
| **端口** | 服务器上的"房间号"。80 号房间专门接待网页访问，22 号房间专门接待远程连接。 |
| **PM2** | 让后端程序在后台一直跑的工具。没有它的话，你关掉 SSH 窗口程序就停了。 |
| **Docker** | 另一种让程序一直跑的方案，比 PM2 更强大。把程序装进"容器"里，和服务器环境隔离开。正式部署推荐用 Docker。 |

### 1.3 需要花钱的地方

| **总计** | **首月约 100-200 元** | |

## 第四部分：ICP 备案（必须做）

### 为什么要备案？

国家规定，在中国大陆服务器上的网站必须登记身份信息。**不备案 = 域名不能用 = 抖音对接不了。**

### 怎么备案？

1. 阿里云控制台搜索 **"ICP 备案"**
2. 点 **"开始备案"**
3. 填写：
   - **主体信息**: 你的真实姓名、身份证号、手机号
   - **网站信息**: 域名、网站名称、用途
4. 上传身份证正反面照片
5. 提交审核

### 要等多久？

- 阿里云初审：1 个工作日
- 管局审核：7-20 个工作日
- **总计约 2-3 周**

> **备案期间不影响开发测试。** 你可以先用 IP 地址访问来测试。建议第一天就提交。

## 第五部分：配置服务器环境（Ubuntu）

### 5.1 连接到服务器

**用阿里云自带的网页终端：**

1. 阿里云控制台 → 轻量应用服务器 → 找到你的服务器
2. 点 **"远程连接"** → **"管理WebTerminal"**
3. 输入用户名 `root` 和你的密码

> 连接成功后会看到一个命令行窗口，下面的命令都在这里输入。
> Ubuntu 用的是 `apt` 包管理器，和 AlmaLinux/CentOS 的 `dnf` 不一样。

### 5.2 安装必要软件

**一行一行复制粘贴，每行输完按回车：**

```bash
# 1. 更新系统
apt update && apt upgrade -y

# 2. 安装 Node.js 22（后端程序运行环境）
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# 3. 安装 Redis（排行榜数据）
apt install -y redis-server
systemctl start redis-server
systemctl enable redis-server

# 4. 安装 Nginx（网页服务器）
apt install -y nginx
systemctl start nginx
systemctl enable nginx

# 5. 安装 Git（下载代码）
apt install -y git

# 6. 安装 Docker（容器化部署，推荐正式环境用，用阿里云源）
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker-aliyun.gpg
chmod a+r /etc/apt/keyrings/docker-aliyun.gpg
echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker-aliyun.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu noble stable" | tee /etc/apt/sources.list.d/docker.list
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
systemctl start docker
systemctl enable docker
```

### 5.3 验证安装

```bash
node --version      # 应该显示 v22.x.x
redis-cli ping      # 应该显示 PONG
nginx -v            # 应该显示 nginx version
docker --version    # 应该显示 Docker version
```

四个命令都有正常输出 = 环境配置成功。

### 5.4 开放防火墙端口

**Ubuntu 用的是 `ufw` 防火墙：**

```bash
# 默认拒绝所有入站连接
ufw default deny incoming

# 默认允许所有出站连接
ufw default allow outgoing

# 开放 22 端口（SSH）
ufw allow ssh

# 开放 80 端口（HTTP 网页）
ufw allow 80/tcp

# 开放 443 端口（HTTPS 安全网页）
ufw allow 443/tcp

# 启用防火墙（注意：确保 SSH 已允许，否则自己会被锁在外面）
ufw --force enable

# 查看已开放的端口，确认 22、80、443 都在
ufw status verbose
```

**同时还要在阿里云控制台开放端口：**

1. 控制台 → 轻量应用服务器 → 你的服务器
2. 点 **"防火墙"** 标签页
3. 点 **"添加规则"**：
   - 应用类型选 **"HTTP(80)"**，点确定
   - 再添加 **"HTTPS(443)"**，点确定
   - SSH(22) 默认已开

> **为什么要开两层端口？** 阿里云安全组是"大楼的门禁"，ufw 是"房间的门锁"。两道都要开，外面的人才能进来。

### 5.5 配置 SSL 证书（备案通过后做）

```bash
# 安装证书工具
apt install -y certbot python3-certbot-nginx

# 申请免费证书（把 yourdomain.com 换成你的域名）
certbot --nginx -d yourdomain.com

# 设置自动续期（证书 90 天过期，这条命令让它自动续）
systemctl enable certbot-renew.timer 2>/dev/null || \
  echo "0 3 * * * certbot renew --quiet --post-hook 'systemctl reload nginx'" | crontab -
```

> 备案没通过之前，可以先不做这步，直接用 http 测试。

## 第六部分：部署项目到服务器

### 6.1 上传代码

**方式一：用 Git（如果代码在 GitHub/Gitee 上）**

```bash
cd /root
git clone 你的仓库地址 raggame
cd raggame/server
```

**方式二：用 SFTP 工具上传**

1. 下载 **WinSCP**（免费）: https://winscp.net
2. 填入服务器 IP、用户名 `root`、密码
3. 把整个 `raggame` 文件夹拖到服务器的 `/root/` 目录

### 6.2 Docker 方式部署（推荐正式环境）

**Docker 会自动完成：安装依赖 → 构建前端 → 启动后端 + Redis + Nginx**

```bash
# 进入 deploy 目录
cd /root/raggame/deploy

# 编辑环境变量文件
cp ../server/.env.example ../server/.env
nano ../server/.env
```

**修改这一行，填入你的 DashScope API Key：**

```
EMBEDDING_API_KEY=sk-xxxxxx  # 换成你在阿里云 DashScope 创建的 API Key
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

```bash
# 配置 Docker 镜像加速器（国内服务器必须做，否则下载镜像会超时）
mkdir -p /etc/docker
cat > /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://hub.rat.dev",
    "https://dockerproxy.com"
  ]
}
EOF

# 重启 Docker 使配置生效
systemctl restart docker

# 验证配置是否生效
docker info | grep -A 5 "Registry Mirrors"

# 一条命令启动所有服务（会自动构建 web 和 server 镜像）
docker compose up -d --build

# 查看运行状态
docker compose ps

# 查看日志确认是否正常
docker compose logs -f
```

看到所有服务状态是 `Up` 就成功了！

### 6.3 PM2 方式部署（适合快速测试）

```bash
cd /root/raggame/server
```

**把 `your_dashscope_api_key_here` 换成你在阿里云 DashScope 创建的 API Key。**

按 `Ctrl+O` 保存，按 `Ctrl+X` 退出。

### 6.4 初始化词库

```bash
cd /root/raggame/server

# 给 tsx 添加执行权限（防止 Permission denied 错误）
chmod +x node_modules/.bin/tsx

# 初始化词库
npx tsx src/db/seed.ts
```

看到 `[Seed] 词库初始化完成，新增 44 个词，总计 44 个词` 就成功了。

> **如果报错 `MODULE_NOT_FOUND`**：说明 node_modules 损坏，执行 `rm -rf node_modules && npm install` 重新安装。

### 6.5 启动后端（两种方式选一个）

**方式一：用 PM2（简单，推荐新手先用这个）**

```bash
# 安装 PM2
npm install -g pm2

# 启动后端
pm2 start "npx tsx src/app.ts" --name raggame-server

# 设置开机自启（服务器重启后自动运行）
pm2 startup
pm2 save

# 查看运行状态
pm2 status

# 查看日志（看有没有报错）
pm2 logs raggame-server
```

**方式二：用 Docker（更专业，推荐正式环境用）**

> Docker 方式已在 6.2 节启动，无需重复操作。如果想单独管理：

```bash
cd /root/raggame/deploy

# 查看所有服务状态
docker compose ps

# 查看后端日志
docker compose logs -f server

# 重启某个服务
docker compose restart server

# 停止所有服务
docker compose down

# 重新启动
docker compose up -d
```

> Docker 方式不需要单独装 Redis，docker-compose 会自动启动一个 Redis 容器。

### 6.6 构建前端页面

**如果用 PM2 方式，需要手动构建前端：**

```bash
cd /root/raggame/web

# 给 vite 添加执行权限（防止 Permission denied 错误）
chmod +x node_modules/.bin/*

# 如果 node_modules 损坏，先重装
rm -rf node_modules package-lock.json
npm install

# 构建生产版本（生成静态网页文件到 dist/ 目录）
npm run build
# 或 npx vite build

# 把网页文件复制到 Nginx 的目录
cp -r dist/* /usr/share/nginx/html/
```

**如果用 Docker 方式，前端已自动构建并部署，无需手动操作！**

> Docker 多阶段构建原理：
> 1. web/Dockerfile 会在容器内执行 `npm install` + `npm run build`
> 2. 生成的 `dist/` 文件通过命名卷 `web-dist` 共享给 nginx 容器
> 3. nginx 直接从这个卷读取前端文件，无需手动 copy

### 6.7 配置 Nginx（PM2 方式需要手动配，Docker 方式已自动配好）

**如果用 PM2 方式，需要手动配置 Nginx：**

```bash
nano /etc/nginx/conf.d/raggame.conf
```

粘贴以下内容（**把 `yourdomain.com` 换成你的域名，没域名之前写 `_`**）：

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # 前端排行榜页面
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 接口
    location /api/ {
        proxy_pass http://127.0.0.1:3000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket 连接（排行榜实时更新）
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
```

保存退出后：

```bash
# 测试配置有没有语法错误
nginx -t

# 没问题的话，重新加载 Nginx
systemctl reload nginx
```

### 6.8 验证部署成功

**用浏览器访问：**

- `http://你的服务器IP` → 应该看到排行榜暗色页面
- `http://你的服务器IP/api/health` → 应该看到 `{"status":"ok"}`
- `http://你的服务器IP/api/words` → 应该看到词库数据

**用命令行检查：**

```bash
# PM2 方式
pm2 status                    # 查看后端状态
curl http://localhost:3000/api/health  # 测试 API

# Docker 方式
docker compose ps             # 查看所有容器状态
docker compose logs -f        # 查看实时日志
curl http://localhost/api/health  # 测试 API
```

都正常 = 部署成功！

## 第七部分：安装 OBS 并配置直播画面

### 7.1 OBS 是什么？

OBS 是**免费**的直播软件。它的作用：

- 把你电脑上的各种画面**拼在一起**（摄像头、网页、图片等）
- 把拼好的画面**发送到抖音**（推流）

```
比喻：
  OBS = 导演
  摄像头 = 1号机位
  排行榜网页 = 2号机位
  OBS 把两个画面拼在一起 → 推给抖音 → 观众看到
```

### 7.2 下载安装

1. 打开 https://obsproject.com
2. 点蓝色 **"Windows"** 按钮下载
3. 双击安装，一路"下一步"
4. 打开 OBS

### 7.3 首次设置

弹出 **"自动配置向导"**：
1. 选 **"仅录制优化"**
2. 分辨率选 `1920x1080`
3. 帧率选 `30`
4. 点"应用设置"

### 7.4 界面长什么样

```
┌──────────────────────────────────────────────┐
│                                              │
│          预览区（大的黑色区域）                 │
│     这里显示观众最终看到的画面                  │
│                                              │
├───────────────┬──────────────────────────────┤
│  场景         │  来源                         │
```

### 7.5 添加摄像头（可选）

1. **"来源"** 框 → 点 **"+"**
2. 选 **"视频采集设备"**
3. 名称填"摄像头"，确定
4. 选你的摄像头设备，确定
5. 预览区出现你的画面
6. **拖动四角**调大小，**拖动中间**移位置
7. 建议放**左边**

### 7.6 添加排行榜网页（核心步骤！）

1. 先在浏览器打开排行榜: `http://你的服务器IP或域名`
2. 回到 OBS → **"来源"** 框 → 点 **"+"**
3. 选 **"浏览器"**
4. 填写：
   - **名称**: `猜词排行榜`
   - **URL**: `http://你的服务器IP或域名`
   - **宽度**: `1920`
   - **高度**: `1080`
   - 勾选 **"当源变为可见时刷新浏览器"**
5. 确定
6. 预览区出现排行榜暗色页面
7. **调整大小和位置**：缩小放到画面**右边**

### 7.7 图层顺序

来源列表里 **上面盖住下面**，正确顺序：

```
1. 猜词排行榜     ← 最上面
2. 摄像头         ← 中间
3. 背景图         ← 最底（如果有）
```

用右侧上下箭头调整。

### 7.8 配置推流到抖音

1. OBS → 右下角 **"设置"**
2. 左侧选 **"直播"**
3. **服务**: 选 **"自定义"**
4. 填入抖音给你的 **推流地址** 和 **推流密钥**（下一节讲怎么获取）
5. 点"应用" → "确定"

## 常见问题 FAQ

### Q1: 备案没通过能测试吗？

能。用服务器 IP 直接访问（`http://47.100.xxx.xxx`），OBS 里也用 IP 添加浏览器源。只是不能对接抖音小程序。

### Q2: 没有小程序，观众怎么参与？

临时方案：让观众在直播间评论"我猜 XXX"，你手动看弹幕。或者把 H5 猜词网页链接分享给观众，手机浏览器打开。

### Q3: OBS 推流后抖音黑屏？

- 检查推流地址和密钥是否正确
- OBS 设置 → 输出 → 编码器试换 **"硬件(NVENC)"** 或 **"软件(x264)"**
- 降低码率到 `2500`

### Q4: Ubuntu 和 CentOS/AlmaLinux 有什么区别？

| | Ubuntu | CentOS | AlmaLinux |
|---|---|---|---|
| 包管理器 | **apt**（本手册用的） | yum | dnf |
| 防火墙 | **ufw**（本手册用的） | firewalld | firewalld |
| 服务管理 | systemctl（一样） | systemctl | systemctl（一样） |
| 关系 | **Debian 系** | 已停更 | CentOS 的继任者 |

> Ubuntu 是最流行的 Linux 发行版，社区大、教程多、遇到问题容易找到解决方案。

### Q5: 阿里大模型 API 收费吗？

DashScope 的 `text-embedding-v3`：
- 新用户 **100 万 tokens 免费**
- 之后约 **0.0007 元/千 tokens**
- 本游戏规模（几百人玩），一个月可能只花几毛钱

### Q6: PM2 和 Docker 该用哪个？

| | PM2 | Docker |
|---|---|---|
| 难度 | 简单，一条命令装好 | 稍复杂，需要配置 docker-compose |
| 效果 | 后台运行 + 自动重启 | 后台运行 + 自动重启 + 环境隔离 + 自动构建前端 |
| 依赖管理 | 需手动安装 Redis、Nginx | docker-compose 自动启动所有服务 |
| 前端部署 | 需手动 build + copy | 多阶段构建自动完成 |
| 推荐 | 新手先验证用 PM2 | **正式部署强烈推荐 Docker** |

**Docker 的优势：**
- ✅ 一键启动：`docker compose up -d` 搞定所有服务
- ✅ 环境隔离：不污染服务器系统
- ✅ 自动构建：前端在容器内 build，无需手动 copy
- ✅ 易于迁移：换个服务器直接复制 deploy 目录就行

两个都能用，选一个就行。**建议先用 PM2 测试，没问题后切换到 Docker 正式部署。**

### Q7: 词库用完了怎么加词？

```bash
cd /root/raggame/server
nano src/db/seed.ts        # 编辑，按格式添加新词
npx tsx src/db/seed.ts     # 重新初始化

# 或
docker compose restart server  # 如果用 Docker
```

### Q9: 常见报错怎么解决？

#### 报错1：`Permission denied`（权限被拒绝）

```bash
# node_modules/.bin/tsx 或 vite 没有执行权限
chmod +x node_modules/.bin/tsx      # server 目录
chmod +x node_modules/.bin/*        # web 目录
```

#### 报错2：`MODULE_NOT_FOUND` 或 `Cannot find module`

```bash
# node_modules 损坏或不完整，重装依赖
rm -rf node_modules package-lock.json
npm install
```

#### 报错3：Docker 镜像下载超时 `i/o timeout`

```bash
# 配置 Docker 镜像加速器（国内服务器必须做）
mkdir -p /etc/docker
cat > /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}
EOF

systemctl restart docker
docker compose up -d --build
```

#### 报错4：防火墙端口没开

```bash
# Ubuntu 用 ufw
ufw allow 80/tcp
ufw allow 443/tcp
ufw status verbose   # 确认 22、80、443 已开放

# 同时要在阿里云控制台 → 安全组里添加规则
```

#### 报错5：SSH 连不上服务器

```bash
# 检查 ufw 是否开放了 SSH
ufw status           # 应该看到 22/tcp ALLOW IN

# 如果没有，手动添加
ufw allow ssh

# 同时确认阿里云安全组开了 22 端口
```

### Q10: 服务器重启了怎么办？
```bash
pm2 startup    # 重新设置开机自启
pm2 save
```

如果用了 Docker：
```bash
cd /root/raggame/deploy
docker compose up -d   # 重新启动
```

Docker 配了 `restart: always` 的话，服务器重启后会自动恢复。
